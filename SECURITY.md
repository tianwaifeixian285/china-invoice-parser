# 安全策略

如果你发现了安全问题，请不要直接公开提交 Issue。

## 报告方式

请通过以下方式私下联系维护者：

- GitHub 私信或仓库所有者公开资料中的联系方式

如果后续项目增加固定安全邮箱，应优先使用安全邮箱。

## 适用范围

当前安全问题通常包括：

- 发票文件处理导致的任意文件读取或写入
- 解析器执行路径中的命令注入
- 样本或发布流程中的敏感信息泄漏
- 验签逻辑被绕过导致错误安全结论

## 处理原则

- 维护者会先确认问题影响范围
- 在修复发布前，尽量避免公开披露细节
- 修复后会在 `CHANGELOG.md` 或发布说明中说明影响版本和修复版本
